üñ•Ô∏è Bash ‚Äî Run Servers in Background

AdaL's bash tool has two modes. Use background mode to start a dev server ‚Äî then MCP can screenshot and iterate without waiting.

‚ö° Normal Mode (blocks)

!npm run build
# Waits until command finishes
# Good for: builds, tests, installs

AdaL waits for completion before responding.

üöÄ Background Mode (non-blocking)

Start the dev server in the background
so I can see the landing page

# AdaL runs: npm run dev (background)
# Returns PID immediately ‚Üí keeps going

Server starts, AdaL continues working.

üîÑ The Iterate Loop ‚Äî Code ‚Üí Run ‚Üí See ‚Üí Fix

1. AdaL starts dev server in background  ‚Üí  localhost:3000 is live
2. Chrome DevTools (MCP) takes a screenshot   ‚Üí  sees the current state
3. AdaL reads server logs for errors     ‚Üí  checks progress output
4. Edits code, screenshot again          ‚Üí  iterate until perfect

üí° Why This Matters

Step Tool What happens
Start server Bash (background) npm run dev ‚Üí PID returned, server runs
Check progress Bash (get output) Read logs: "compiled successfully on :3000"
See the page Chrome DevTools (MCP) Screenshot localhost:3000 ‚Üí AI sees result
Fix & repeat AdaL edits files Hot-reload updates page ‚Üí screenshot again

üíé Class Step 2 ‚Äî The 20% Polish

This is where you act as the Art Director to help AdaL push the clone from "looks close" to 100% pixel-perfect.

ü§ñ AdaL's Job: Deep DOM & Two-Tab Verification

AdaL drives the browser automation to close the visual gap:

  • Opens the reference site and your localhost side-by-side.
  • Performs the "Stitch Test" to hunt for micro-deviations.
  • Uses Deep DOM Interrogation (Sniper CSS) to extract exact multi-stop gradients, multi-layer box shadows, and WebGL <canvas> loops.

üëÄ Your Job: Art Director & QA

Observe AdaL's process and provide corrective visual feedback.

"The purple orbit is a muddy blob, you used too much blur. Check the exact linear-gradient."

If AdaL accidentally approximates a glow or misinterprets a layout gap, point it out. Your job is to validate the agent's logic against your own "Eye Test".
